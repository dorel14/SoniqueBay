## Points d'Attention pour la Migration

1. **Communication Celery** : Le test `test_celery_communication.py` valide les échanges entre workers. Il doit être adapté pour TaskIQ.
2. **Flux Complets** : Les tests d'intégration valident des scénarios end-to-end (scan → extraction → insertion).
3. **Redis** : Ces tests utilisent Redis pour la communication inter-services.

## Notes

1. **Environnement** : Ces tests nécessitent un environnement Docker complet (API, Redis, PostgreSQL).
2. **Isolation** : Chaque test doit être isolé pour éviter les effets de bord.
3. **Rollback** : En cas d'échec, les données de test doivent être nettoyées.
